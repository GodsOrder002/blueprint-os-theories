对，**MariaDB** 当然应该在清单里，而且在你这个“电脑端为主、最好手机/iPad也能管理”的口径下，它其实是很常见的一条主线（MySQL 生态、离线可跑、管理工具选择多）。

我把“数据库 + 管理工具（全开源）”按你要的维度**重新提炼成一张清单**：电脑端为主，分 **原生 GUI / Web UI（手机友好）**；并把 MariaDB 补齐。

---

## 电脑端（数据库 + 管理工具，全开源）

### 1) SQLite（单文件库）

* 原生 GUI：DB Browser for SQLite、SQLiteStudio
* Web UI（手机/iPad 友好）：Datasette、Adminer

### 2) PostgreSQL（多人/权限/审计更顺）

* 原生 GUI：pgAdmin、DBeaver Community
* Web UI（手机/iPad 友好）：CloudBeaver、Adminer

### 3) MariaDB（MySQL 生态）

* 原生 GUI：DBeaver Community、HeidiSQL
* Web UI（手机/iPad 友好）：phpMyAdmin、Adminer、CloudBeaver

---

## 手机/iPad “最好适配”的结论

你要的“最好适配手机/iPad”**最稳的解**不是在手机上装数据库，而是：
**电脑上跑数据库 + 电脑上跑 Web 管理工具**，然后手机/iPad 用浏览器访问（离线也行，只要同一局域网/热点）。

对应的 Web 管理工具就是上面那几个：CloudBeaver / Adminer / phpMyAdmin / Datasette（SQLite 专用）。
